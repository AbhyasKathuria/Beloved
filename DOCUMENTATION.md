# Technical Documentation — Beloved 🌹✨

**Beloved** is a private, one-to-one digital bouquet and love letter gifting web application. Senders can design a custom flower bouquet by freely placing and arranging illustrations on a canvas, compose a personal love letter, record a voice note, and generate a secure sharing link. When the recipient opens the link, the bouquet blooms onto their screen, the envelope slides open to reveal the letter, and the voice cassette player plays their voice note.

---

## 📖 The Story Behind the App
Inspired by a real-life long-distance relationship, Beloved was created from a simple wish: a girl who adores flowers, and a boy who wanted to make her smile across the miles. 

Because real flowers wither and emojis feel too small, Beloved was designed to create digital bouquets that **never fade, never wilt, and carry love across any distance.**

## 🌟 Key Features & Engineering Implementations

### 1. Real-Time Directory Watcher Plugin (`vite.config.ts`)
To make editing fluid, a custom file-system watcher was built directly into the Vite development config:
* It monitors the root `/Flowers` directory for additions, deletions, or renames.
* It uses a debounced trigger to execute `generate-flower-list.js`.
* It updates `src/flowerList.json` which instantly triggers Vite's Hot Module Replacement (HMR) to refresh the browser picker panel in under **100ms** without resetting the dev server.
* The watcher uses path prefix checks (`filePath.startsWith`) to avoid triggering infinite file-write loops.

### 2. Transparent Background Clipper (`imageLoader.ts`)
To allow users to paste standard illustrations without tedious manual preprocessing, the client features a pixel-level background keyer:
* Reads the image source and draws it to a temporary HTML5 Canvas.
* Scans the RGBA byte buffer of the image.
* Any pixel where `Red`, `Green`, and `Blue` are all above `240` (near-white backgrounds) is modified to `Alpha = 0` (transparent).
* Returns a Base64 data URL with anti-aliased, soft-blended boundaries.

### 3. V-Shape Bouquet Guides (`Canvas.tsx`)
* **Design Helper:** During editing, a dashed peachy gradient outline representing wrapping paper and flower stems is rendered on the canvas layer, guiding users to place flowers in a traditional cone arrangement.
* **Smart Hiding:** In `RevealPage.tsx`, the canvas is rendered in `readOnly` mode. The peachy envelope wrapper guide is hidden, leaving **only** the flowers to bloom in a floating, magical sequence.

### 4. Horizontal Mirror Flipping (`CanvasControls.tsx` & `Canvas.tsx`)
* Select any flower and click **Mirror / Flip** to toggle its `scaleX` value between positive and negative (e.g., `0.7` to `-0.7`).
* By anchor-offsetting the image to its exact center (`width / 2`, `height / 2`), the mirroring occurs in place without position shifting.

### 5. StrictMode Double-Mount Protection (`RevealPage.tsx`)
In React's development mode (Strict Mode), components mount twice. To prevent race conditions:
* The mounting hook returns a cleanup callback that clears running intervals (`clearInterval`) and timeouts (`clearTimeout`).
* A state-tracking flag (`active`) cancels database resolution updates if the user leaves the page before the API responds.
* Added bounds checks (`currentIndex < allFlowers.length`) to prevent appending `undefined` elements to the Canvas which would otherwise crash Konva rendering.

## ⚡ Production Deployment (Vercel)

### Step 1: SQL Editor Schema Setup
Go to your Supabase SQL Editor and run the following queries to create the table, enable Row-Level Security, and configure policies:
```sql
-- 1. Create table
create table bouquets (
  id uuid default gen_random_uuid() primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  sender_name text not null,
  recipient_name text not null,
  occasion text not null,
  letter_content text not null,
  canvas_data jsonb not null,
  snapshot_url text,
  audio_url text,
  expires_at timestamp with time zone,
  view_count integer default 0 not null,
  max_views integer
);

-- 2. RLS Policies
alter table bouquets enable row level security;
create policy "Allow public inserts" on bouquets for insert with check (true);
create policy "Allow public select" on bouquets for select using (true);
create policy "Allow public updates to view count" on bouquets for update using (true);
```

### Step 2: Storage Bucket Security Policies
Create a public bucket named `bouquets`, go to **SQL Editor**, and run this to allow anonymous uploads and downloads:
```sql
-- Allow public inserts (uploads)
create policy "Allow public uploads to bouquets"
on storage.objects for insert
to public
with check (bucket_id = 'bouquets');

-- Allow public selections (downloads)
create policy "Allow public read from bouquets"
on storage.objects for select
to public
using (bucket_id = 'bouquets');
```

### Step 3: Vercel Settings
Link your GitHub repository to Vercel and input the following **Environment Variables**:
* `VITE_SUPABASE_URL` = `https://your-project-id.supabase.co`
* `VITE_SUPABASE_ANON_KEY` = `[Your public anon publishable key]`
